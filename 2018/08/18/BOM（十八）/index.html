Doctype

（1）渲染模式：标准模式，混杂模式。
在多年以前（IE6诞生以前），各浏览器都处于比较封闭的发展中，不考虑兼容性，随着web的发展，兼容性问题的解决越来越显得迫切，随即，各浏览器厂商发布了按照标准模式（遵循各厂商制定的统一标准）工作的浏览器，比如IE6就是其中之一，但是考虑到以前的网站并不支持标准模式，所以各浏览器在加入标准模式的同时也保留了混杂模式（即以前那种未按照统一标准工作的模式，也叫怪异模式）
（2）三种标准模式的写法：当在HTML文件的第一行写一下代码，则开启标准模式，不写，则是怪异模式。
①，<!DOCTYPE html>
②，<!DOCTYPE HTML PUBLIC “-//W3C//DTD HTML 4.01//EN” “http://www.w3.org/TR/html4/strict.dtd">
③，<!DOCTYPE html PUBLIC “-//W3C//DTD XHTML 1.0 Strict//EN” “http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

BOM：Browser Object Model，浏览器对象模型，定义了操作了浏览器的接口

（1）BOM对象：Window，History，Navigator，Screen，Location等。
（2）由于浏览器厂商的不同，BOM对象的兼容性极低，一般情况下，我们只用其中部分功能。

window对象的方法

①close()：将关闭有window指定的顶层浏览器窗口，某个窗口可以通过调用self.close()或只调用close()来关闭其自身，只要通过JavaScript代码打开的窗口才可以有JavaScript代码关闭，这阻止了恶意脚本终止用户的浏览器。
②comfirm()：显示一个带有指定消息和OK及取消的按钮的对话框。comfirm(message)中的message指要在window上弹出的对话框显示的文本。按下OK及取消时，返回true或者false。据此，可以进行下一步的操作。
③prompt()：显示可提示用户输入的对话框，括号中可以输入两个参数，prompt(text,defaultText)其中text指的是要在对话框中显示的纯文本，后面的参数指的是默认的输入文本。如果用户单击提示框的取消按钮，则返回null，如果单击确认按钮，则返回的是当前输入框中的文本。

navigator对象属性

①userAgent：返回有客户机发送服务器的user-agent头部的值。调用的时候navigator.userAgent。用以匹配解决兼容性问题，参考can I use网站。
②online：返回指明系统是否处于脱机模式的布尔值。（实际上就是判断是否联网的状态，假如有网，返回true，没网就返回false）用以判断点击链接时走网络请求还是本地缓存。
③appCodeName：返回浏览器的代码名，所有的浏览器都是Mozilla。

History对象

表示窗口的浏览历史，但出于隐私方面的原因，History对象不再允许脚本访问已经访问过的实际URL。
（1）History对象属性：
length：返回浏览器历史列表中的URL数量
（2）History对象方法：
①back()：加载history列表中的前一个URL。
②forward()：加载history列表中的下一个URL。
③go()：加载history列表中的某个具体的页面，中间可以填数字，正，为向前加载，但假如超过URL列表，返回undefined，为负，则向前加载，同理，超过URL列表，则返回undefined。

location对象

包含有关当前的URL信息，是window对象的一部分，可通过window.location属性来访问。
（1）location对象属性：
①href：设置或者返回完整的URL，当给href赋值时，可以跳转到当前URL中的页面中去。
②hash：设置或者返回URL后面的#号后面的值（加#号）
③host：设置或者返回主机名或者当前URL的端口号（http协议默认80端口，返回的也是域名）
④hostname：设置或者返回当前的URL主机名。
⑤pathname：设置或返回当前URL的路径部分（即URL中域名后的浅色字体）
⑥port：设置或返回当前的URL的端口号，假如是默认的80端口，则返回空。
⑦protocol：设置或返回当前的URL的协议（http或其他）
⑧search：设置或返回从文号开始的URL部分（查询部分，传输给后台的信息部分）
（2）location对象方法：
①assign()：加载新的文档（括号中填写链接的地址，在当前页面打开链接网址，和window.open()方法的区别在于后者是在一个新的标签页上打开网址）
②reload()：重新加载当前文档（刷新页面)
③replace()：用新的文档替换当前文档
